MCP de Railway: local vs remoto

El MCP es el puente que deja a un agente de IA operar tu Railway (desplegar, ver logs, cambiar variables). Hay dos formas de conectarlo.

Recomendado para nosotros
Local
Corre en tu máquina, vía el CLI

El agente usa el CLI de Railway que ya tienes instalado. Más poder, menos fricción de login.

railway setup agent
Remoto
Hospedado por Railway en la nube

El agente se conecta a un endpoint de Railway. Sin instalar nada local.

railway setup agent --remote
¿Cuál elegir? Flujo de decisión
¿Necesitas logs, variables o dominios desde el agente? Local
¿Quieres cero instalación y tokens revocables? Remoto
¿Quieres un agente de IA que analice logs y depure? Remoto
